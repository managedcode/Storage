# UI/UX Engineering

UI/UX guidance in MCAF ties user outcomes to implementable engineering decisions.

## Core Concerns

- usability
- accessibility
- design-system consistency
- handoff between design and engineering
- front-end maintainability

## Team Rules

- start from user goals and constraints
- treat accessibility as a baseline quality attribute
- keep tokens, spacing, and interaction rules explicit
- avoid ad hoc component variants that bypass the system
